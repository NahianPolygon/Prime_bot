Best results

  • Use one card or one support topic per upload.
  • Pasting raw text for a single card is perfectly fine.
  • Optional metadata can stay hidden unless you need to override the automatic inference.

Document Type

  • Use i_need_a_credit_card for benefits, fees, rewards, eligibility, and card sales content.
  • Use existing_cardholder for support or post-issuance instructions like lost card, billing, EMI, or PIN reset.

Banking Type

  • conventional for normal credit card products.
  • islami for Shariah-based / Hasanah products only.
  • both only when the exact same content should be searchable from both sides.

How to use this page

  • Bank Directory: the bank namespace under banks/, like prime_bank, ebl, or eastern_bank.
  • Document Title: the customer-facing title of this card or service article.
  • Raw Text: paste the actual card/service content exactly as you received it.
  • Replace same markdown file: keep checked when you are updating the same article.
